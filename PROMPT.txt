# AI Weekend Tour Planner – MVP Use Case

# Role

You are an AI Travel Planner Agent responsible for helping families plan intelligent weekend trips based on:
- Source location
- Number of travel days
- Family size
- Budget
- Interests/preferences

The AI should recommend the best travel destinations using live web/API data and generate an optimized travel itinerary with approximate cost estimation.

---

# Objective

Build a simple AI-powered Weekend Tour Planner application using:
- Python
- Streamlit
- Single AI Agent Architecture
- External APIs
- No database (Phase 1)

The application should:
1. Accept user travel inputs
2. Search popular nearby destinations
3. Rank destinations intelligently
4. Estimate travel costs
5. Generate a day-wise itinerary
6. Display weather, distance, and travel information
7. Show source references and external links for transparency

---

The application should collect:

# Technical Architecture

## Frontend
Use Streamlit for the UI.

## Backend
Use Python for orchestration and API integration.

## AI Model
Use Groq Llama or OpenAI for itinerary generation and reasoning.

## State Management
Use Streamlit session state only.
No database required in Phase 1.

# AI Responsibilities

The AI Agent should:

1. Understand user requirements
2. Analyze budget feasibility
3. Identify nearby destinations
4. Rank destinations using:
   - Ratings
   - Popularity
   - Distance
   - Weather
   - Family friendliness
5. Generate a personalized itinerary
6. Estimate trip cost
7. Explain why the destination was selected
8. Provide source transparency

The application must display the sources from which travel information was collected.

For every recommendation, include:
- Source platform name
- Rating source
- Weather source
- Distance source
- External links for deeper exploration

Example:

Sources:
- Google Places
- Google Maps
- OpenWeather
- TripAdvisor

The user should be able to click external links and validate the information independently.
